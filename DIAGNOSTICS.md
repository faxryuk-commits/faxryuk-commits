# Диагностика проблем с парсерами

## Проверка логов на Railway

1. Зайдите в Railway Dashboard
2. Откройте ваш проект
3. Перейдите во вкладку "Deployments" → выберите последний деплой → "View Logs"

## Что искать в логах:

### 1. Запуск парсера
Ищите строки типа:
```
INFO: Начинаю парсинг Wildberries для запроса: телефон
INFO: Парсер вернул X товаров
```

### 2. Если товары не найдены
Ищите строки:
```
WARNING: Парсер вернул пустой список товаров
WARNING: Карточки товаров не найдены
```

### 3. Проблемы с селекторами
Если видите:
```
INFO: Найдено 0 карточек товаров используя селектор
WARNING: Стандартные селекторы не сработали
```
Это означает, что структура HTML изменилась.

### 4. Проблемы с валидацией
Если видите:
```
WARNING: Товар не прошел валидацию: name=True, url=True, source=True
```
Проверьте, какие поля отсутствуют.

## Возможные решения:

### Вариант 1: Использовать Selenium для JavaScript-сайтов
Если сайты используют динамическую загрузку контента.

### Вариант 2: Использовать официальные API
- Wildberries имеет API
- Uzum может иметь API

### Вариант 3: Обновить селекторы
Если структура HTML изменилась, нужно обновить селекторы в парсерах.
